# 梦山 资源规范 / Asset Specification

> **引擎**: Unreal Engine 5
> **风格**: 厚涂风格化 3D (Painterly Stylized 3D)
> **核心依赖**: [资源表现总则](./00_资源表现总则.md)、[世界观综述](../06_世界观/00_世界观综述.md)

---

## 1. 概述 / Overview

本规范定义梦山项目在 UE5 下的资源命名、技术规格与厚涂风格化 3D 的制作要求。所有角色、场景、UI、音效等资产 **必须** 符合本文档。

## 2. 命名规范 / Naming Convention

### 2.1 基本格式

```
[前缀]_[类别]_[名称]_[变体]_[序号]
```

### 2.2 前缀定义

| 前缀 | 资源类型 | 示例 |
| :--- | :--- | :--- |
| `tex` | 纹理 / Texture | `tex_char_traveler_base_d` |
| `mat` | 材质 / Material | `mat_env_mountain_rock` |
| `mdl` | 模型 / Model | `mdl_char_traveler_lod0` |
| `skm` | 骨骼网格 / Skeletal Mesh | `skm_char_traveler_body` |
| `anim` | 动画 / Animation | `anim_char_climb_idle` |
| `vfx` | 特效 / VFX | `vfx_coop_lift_success` |
| `sfx` | 音效 / SFX | `sfx_ui_click_001` |
| `bgm` | 背景音乐 / BGM | `bgm_scene_mountain_path` |
| `ico` | 图标 / Icon | `ico_ui_btn_coop` |
| `wbp` | 控件蓝图 / Widget Blueprint | `wbp_ui_main_menu` |

### 2.3 类别定义

| 类别 | 说明 | 子类别 |
| :--- | :--- | :--- |
| `char` | 角色 | traveler, npc, buddy |
| `prop` | 道具/可交互物 | hold, rope, checkpoint |
| `env` | 环境 | terrain, rock, vegetation, building |
| `ui` | 界面 | btn, panel, icon, frame |
| `coop` | 协作相关 | lift, pull, assist |
| `scene` | 场景 | mountain_path, summit |

## 3. 厚涂风格化 3D 技术规格 / Painterly Stylized 3D Specs

### 3.1 纹理规格

| 用途 | 分辨率 | 格式 | 风格要求 |
| :--- | :--- | :--- | :--- |
| 角色主贴图 | 2048×2048 | PNG/TGA → UE Texture | 手绘厚涂，笔触可见，色块分明 |
| 角色法线（可选） | 2048×2048 | PNG | 柔和，辅助体积感，非写实雕刻 |
| 场景主贴图 | 1024–2048 | PNG/TGA | 手绘，有机纹理，避免照片感 |
| UI 图标 | 128–256 | PNG | 手绘风格，与 3D 一致 |
| UI 背景/面板 | 512–1024 | PNG | 柔和渐变，可平铺 |
| 特效贴图 | 256–512 | PNG | 软边、透明度渐变 |

### 3.2 材质与着色

*   **基础原则**: 厚涂感通过 **手绘贴图** 为主；Shader 辅助光照与边缘。
*   **光照响应**: **必须** 柔和；可考虑 Toon/Cel-shading 或自定义 NPR 光照模型。
*   **禁止**: 强 PBR 金属/粗糙度写实感；高光过强、硬边反射。
*   **推荐**: 柔和高光、软阴影、轻微环境光遮蔽以增强体块感。

### 3.3 模型规格

| 用途 | 面数 (Triangles) | 骨骼数 | LOD | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| 主角/旅者 | ≤25,000 | ≤60 | 3 级 | 攀爬动作需足量骨骼 |
| 搭档/NPC | ≤15,000 | ≤50 | 2 级 | - |
| 环境物件 | ≤8,000 | - | 2 级 | 岩石、植被、建筑 |
| 协作道具 | ≤3,000 | 按需 | 1–2 级 | 支点、绳索等 |
| 远景/装饰 | ≤2,000 | - | 1 级 | 低模、可批量实例化 |

### 3.4 动画规格

| 类型 | 帧率 | 说明 |
| :--- | :--- | :--- |
| 攀爬动作 | 30 fps | 抓取、释放、借力、托举、拉拽 |
| 待机/循环 | 30 fps | 需平滑循环，无明显跳帧 |
| 协作反馈 | 30 fps | 成功/失败表情、手势，时长 ≤2s |
| UI 动效 | 60 fps | 轻量，优先在 UMG 内实现 |

## 4. 音频规格 / Audio Specs

| 类型 | 采样率 | 格式 | 说明 |
| :--- | :--- | :--- | :--- |
| BGM | 44.1 kHz | OGG | 循环 2–5 分钟；温暖、希望感 |
| SFX | 44.1 kHz | WAV/OGG | 时长 ≤5 秒；协作成功/失败需区分 |
| 环境音 | 44.1 kHz | OGG | 风声、鸟鸣等；可循环 |
| UI 音效 | 44.1 kHz | WAV | 点击、打开、关闭；柔和不刺耳 |

## 6. 交付清单 / Delivery Checklist

### 模型

- [ ] FBX 导出，轴向与 UE5 一致（Y-up）
- [ ] 含骨骼与蒙皮（如适用）
- [ ] LOD 模型（按规格）
- [ ] 导入 UE5 测试通过

### 纹理

- [ ] 源文件（PSD 等）保留
- [ ] 导出 PNG/TGA，符合分辨率规范
- [ ] 厚涂风格一致，无照片素材混入

### 动画

- [ ] FBX 导出，骨骼命名一致
- [ ] 片段拆分清晰（idle、climb、coop 等）
- [ ] 循环动画无跳帧
- [ ] 根骨骼/位移按 UE5 需求处理

### 音频

- [ ] 工程文件备份
- [ ] 导出格式正确
- [ ] 音量标准化
- [ ] 无杂音、爆音
- [ ] 风格符合「温暖、通透」基调
