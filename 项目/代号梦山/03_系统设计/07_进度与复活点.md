# 梦山 系统设计规范 — 进度与复活点 (Progress & Respawn System Spec)

> **设计目的 (Design Goal)**  
> 定义检查点、营地/祭坛复活、稀有复活道具等复活途径与进度保存；与状态系统（存活/待救援/死亡）及道具系统对接。

> **关联文档**: [状态系统](04_状态系统.md)、[道具系统](05_道具系统.md)、[环境与关卡](03_环境与关卡.md)

## 1. 本规范覆盖范围

*   **复活途径（来自原《状态系统》描述，不做设计改动）**：
    * **营地/祭坛复活点**：在特定检查点或营地使用祭坛/营火，使死亡队友回归。
    * **稀有复活道具**：在道具系统中定义稀有复活类道具，可在特定条件下将死亡队友拉回存活状态。
*   **检查点**：玩家到达检查点后的进度保存与复活点绑定，由关卡与进度配置决定；具体实现随项目细化。
*   **待扩展**：祭坛/营火的数量与分布、复活道具的关系门控（如是否需 2 同行者）、地图重置或 Roguelite 规则等，随关卡与数值文档补充。

### 1.1 PEAK 基础设计还原（参考）

*   **祭坛 (Altar)**：PEAK 中祭坛位于营地篝火旁（各 biome 末端），与篝火绑定；交互后**全队复活**，无需消耗道具。与梦山「营地/祭坛复活点」一致。
*   **Scout Effigy（侦察兵雕像/复活雕像）**：PEAK 中为稀有、**单次使用**道具，可从行李等途径获得；使用后可在**原地复活一名**已死亡队友。与梦山「稀有复活道具」一致；梦山可保留关系门控（如是否需 2 同行者才可使用）。
*   **检查点/营地**：PEAK 以营地篝火为安全点与复活锚点；梦山检查点与复活绑定规则可对齐。

## 2. 与系统架构的映射

| 系统 | 本规范覆盖 |
| :--- | :--- |
| [SYS_PROGRESS] 进度与复活点 | 检查点、营地/祭坛复活、复活道具触发的复活途径 |
| [SYS_STATE] 状态系统 | 死亡 → 复活后回归存活状态；见《状态系统》3.5 |
| [SYS_ITEM] 道具系统 | 复活类道具的定义与使用规则；见《道具系统》 |
| [SYS_ENV] 环境与关卡 | 营地、祭坛、检查点位置；见《环境与关卡》 |
