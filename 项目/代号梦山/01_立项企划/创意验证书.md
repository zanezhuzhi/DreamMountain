## 1. 创意原点 (The Spark)

*   **原始描述**:
    *   切入《光遇》2.6 亿流失用户市场，用《Peak》验证过的「强合作攀岩玩法」解决社交游戏缺乏游戏性的痛点，并在中期后优化《光遇》缺乏双人玩法的困境。
*   **出发角度**: 受众 + 玩法 + 市场缺口

---

## 2. 第一性原理挖掘 (First Principles Excavation)

### 2.1 体验真值 (Truth of Experience)
剥离掉攀岩、山峰、美术风格后，玩家**真正渴望**的是：**在不确定的旅途中，因一次真实的互助而与陌生人建立信任，并最终拥有「固定的人」一起经历预设的双人/多人情境**。即：从「偶然的善意」到「可预期的羁绊」。

**设计前提澄清**：
*   **光遇最成功之处**：把「从陌生人成为好朋友」做到极致；梦山关注光遇与 Peak 各自的成功点，**组队/双人体验的前提正是光遇式社交带来的挚友关系**——即 Peak 式协作发生在「已经愿意一起玩的人」之间。
*   **前期以单人为主**：多数玩家前期以**单人**体验为主。在此历程中，陌生人提供**友善但非必需**的互助（类似光遇：可帮可不帮，不绑架进度）。
*   **保护社恐与独行体验**：不强制社交、不剥夺「一个人也能完整玩」的孤独感；社恐与偏好独处的玩家应被保护，可选地接受善意而非被系统推向组队。

### 2.2 需求验证 (Demand Validation)
*   **目标受众 (Target Audience)**: 光遇流失用户（尤其大学离校后、25–35 岁因内容幼稚与玩法不足而流失）、偏好情感与社交但不满「只有跑图/收集」的玩家；渴望有「一起做事」而非「一起挂机」的轻度 MMORPG 用户。
*   **竞品现状 (Market Context)**:
    *   《光遇》：强情感与社交氛围，但核心玩法偏跑图/收集，游戏性不足；双人/多人情境薄弱；对男性与 25–35 岁筛选过重，流失用户规模大（详见 2.2.1 流失归因）。
    *   《Peak》：验证了「强合作攀岩」的可玩性与协作手感，但非情感向 MMORPG。
*   **分析**: 社交游戏缺乏「必须协作才能过关」的游戏性，情感游戏缺乏「双人专属内容」的长期锚点。梦山用攀岩+互助把「协作」做成核心玩法，用预设双人/多人情境解决关系沉淀。

#### 2.2.1 受众可触达性与转化假设

**光遇用户画像（参考）**:
| 维度 | 光遇特征 | 备注 |
| 性别 | 男性不足 20%，女性约 80% | 远低于大盘（大盘约男 55% / 女 45%） |
| 年龄 | 25 岁以下超 90%，30 岁以上不足 5% | 极度年轻化 |
| 职业 | 高中生 + 大学生合计超 65% | 学生为主体 |
| 地域 | 新一线、三线城市占比相对较高 | |

**流失归因假设（核心论点）**:
*   光遇用户随着年龄自然增大（尤其是大学离校后），因【**无法提供更进一步的多人体验**】且【**脱离较幼稚的内容匹配期**】，造成**断崖式流失**。
*   **关键推论**：这批人**愿意尝试**此类情感社交游戏，只是光遇内容偏幼稚、玩法缺乏深度，不再合适。即：**需求仍在，供给不匹配**。

**数据支撑**:
*   留存调查：愿意留下因「朋友/伴侣一起玩」占 32%，但「玩法没有做任何调整与跟进」；社交乐趣随游戏时长递增（1 年半–2 年达 54%），探索/挑战乐趣递减。
*   流失问卷：8 个月用户进入「养老期」；**63% 流失原因为「缺乏长线玩法和内容更新」**。
*   市场特征：对男性与中青年（25–35）玩家筛选过重；核心 gameplay 偏薄弱；艺术风格偏卡通；收入存在较大提升空间。
*   赛道判断：非暴力的情感交互赛道远未饱和。

**对梦山的战略含义**:
*   **机会窗口**：流失用户非「不再要情感社交」，而是渴求「更成熟的多人体验」与「更有深度的玩法」。梦山用强协作攀岩 + 双人/多人情境锚点，可直接回应此缺口。
*   **风险收敛**：若流失主因是「不再想要此类情感社交」，则梦山无法召回；现有假设指向**相反结论**，需在 Demo 阶段验证：25–35 岁、曾为光遇用户的群体对梦山原型的接受度。

#### 2.2.2 攀岩 vs 其他协作形式的选择理由 *(新增)*
*   **为何攀岩，而非解谜/战斗/建造？**
    *   攀岩：物理具象、节奏可控、单点协作清晰（托举/拉拽），失败即「重试」而非「死亡」，挫败感可控；与「旅途、互助、共同克服障碍」的隐喻高度契合。
    *   解谜：偏智力，协作点分散，难以形成「你托我、我拉你」的具身化信任感。
    *   战斗：易引入竞争/数值，偏离「纯互助」情感基调；且光遇用户对战斗接受度存疑。
    *   建造：节奏偏慢，难以在单次会话内形成「我们一起做到了」的高峰体验。
*   **Peak 受众 vs 光遇受众重叠度**: [待补充：两者用户画像重叠假设，及对设计的影响]
*   **失败耐受度**: 强协作意味着「配合不好会卡关」。光遇用户多为轻度情感向，需在 Demo 中验证：失败惩罚强度、重试成本、挫败感是否在可接受范围内。[待补充：Demo 验证结论]

#### 2.2.3 与光遇的明确差异

**光遇痛点与梦山应对**:
| 光遇痛点 | 梦山应对 |
| :--- | :--- |
| **前期**: 缺少叙事，独狼与轻社交玩家难以进入；追求无文字表达抬高上手门槛 | **强化叙事**；保留孤独者观光与心理按摩调性，但不牺牲必要引导；攀登天生目标感降低迷失 |
| **中后期**: 缺少玩法与内容，游戏性、趣味性、目标感不足 | **强化 gameplay**：攀爬过关即核心玩法，每次操作有明确成败；双人强互动关卡（参考 It Takes Two 等）；收集/合成与物资流转可拓展 |
| **创作者坚持与玩家需求冲突** | 全盘延续核心设定（情感驱动社交 + 艺术化表达），在保持调性前提下补足玩法与内容缺口 |

**继承与差异化**:
*   **继承**: 陌生人→挚友的社交体验、孤独者的观光之旅与心理按摩调性、情感驱动社交 + 艺术化表达。
*   **差异化**: 用「协作过关」取代「协作陪伴」；用「双人强互动关卡」取代弱绑定的一起跑图；用「机制锚定的双人情境」取代自发维系；用「天生目标感（攀登）」缓解无文字表达带来的迷失与门槛。

*   **社交疲劳风险**: 强协作伴随沟通与配合成本。需验证：玩家是否会因「必须认真配合」而产生疲惫感。[待补充：设计缓解手段，如异步协作、降低沟通门槛]

## 3. 媒介契合度 (Medium Fit)

### 3.1 交互必要性 (Interaction Necessity)
为什么必须通过**电子游戏**实现？
*   **分析**: 电影/小说中的「互助」是旁观；只有让玩家**亲自操作攀爬、亲自在关键节点依赖他人或被依赖**，才能产生「我们一起做到了」的归属感。陌生人变挚友的体验依赖**实时协作的成败**与**共同目标**，只能由交互承载。

### 3.2 可实现性边界 (Feasibility)
*   **技术**: Peak 已验证攀岩+协作手感；光遇已验证大世界情感社交与多平台。两者结合在技术上有先例。
*   **风险**: 攀岩物理与网络同步、陌生人匹配与信任建立的节奏设计，需在 Demo 阶段验证。

#### 3.2.0 Demo 范围界定 *(新增)*
Demo 圈定以下四个支柱，作为验证与开发的范围边界：

| 支柱 | 范围 | 说明 |
| :--- | :--- | :--- |
| **1. 独立登山** | 物理模拟下的单人登山过程 | 玩家在物理模拟下独立完成登山，验证攀爬手感与单人可玩性 |
| **2. 偶遇陌生人** | 与陌生玩家的偶遇与初接触 | 玩家在旅途中偶遇其他陌生玩家，验证偶遇机制与初识体验 |
| **3. 玩家互动** | 玩家间的协作与互动 | 托举、拉拽、借力等互助行为，验证协作手感与关系建立节奏 |
| **4. 关卡细化** | 登山关卡的路线与节点设计 | 路线拓扑、支点、协作节点、检查点等关卡的细化设计 |

**逻辑依赖**: 独立登山（1）与关卡细化（4）奠定基础可玩性；偶遇陌生人（2）与玩家互动（3）在此之上验证社交与协作体验。

Demo 必须验证以下假设，任一失败则需重构或终止：

| 序号 | 验证项 | 通过标准 | 状态 |

| 2 | 网络同步 | 弱网/高延迟（如 150ms+）下协作仍可完成，无频繁断线或动作错位 | [待验证] |

## 4. 结论 (Verdict)

*   **决策**: GO（进入概念设计）
*   **核心定义 (The Core Definition)**:
    *   **通过旅途中基于攀岩与互助的协作玩法，交付「陌生人因互助结识为游戏中挚友，并进入预设双人/多人社交情境」的情感与社交体验。**
    *   **设计边界**：前期以单人体验为主；陌生人互助为友善且非必需（学光遇）；组队与双人内容建立在「光遇式社交带来挚友」的前提上；保护社恐与独行玩家的孤独感，不强制社交。
