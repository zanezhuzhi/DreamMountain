> **文档状态**: 初稿

## 1. 顶层定义 (High-Level Definition)

### 1.1 核心体验 (Core Aesthetic)
*   **继承定义**: 通过旅途中基于攀岩与互助的协作玩法，交付「陌生人因互助结识为游戏中挚友，并进入预设双人/多人社交情境」的情感与社交体验。
*   **设计边界**: 前期以单人体验为主；陌生人互助为友善且非必需；组队与双人内容建立在「光遇式社交带来挚友」的前提上；保护社恐与独行玩家的孤独感。

### 1.2 产品愿景 (Product Vision)
> 在通往山顶的旅途中，通过必须协作的攀岩与互助，与陌生人成为挚友，并一起进入只属于彼此的双人世界。

---

## 2. 设计支柱 (Design Pillars)

### 支柱 1: 互助即玩法 (Cooperation as Gameplay)
*   **原则**: 协作不是附加功能，而是过关与成长的核心手段；攀爬与互助必须有明确的操作与成败反馈。
*   **意味着 (We Do)**: 主线路段保证单人可通关；在隐藏/分支设计「单人难以完成、协作才有解法」的关卡；托举、拉拽、借力等操作有清晰输入与反馈。
*   **不意味着 (We Don't)**: 主线强制多人协作；协作做成纯社交动作（仅表情/站桩）；主线单人轻松过关而协作失去意义。

### 支柱 2: 情感优先于数值 (Emotion Over Numbers)
*   **原则**: 核心体验是「信任感、归属感、一起完成一件事」，而非战力、排行榜或付费强度。
*   **意味着 (We Do)**: 用情境、演出、双人/多人专属内容强化关系体验；成长与解锁服务于「更多一起做的事」。
*   **不意味着 (We Don't)**: 付费购买协作能力或攀爬数值优势；用排行榜/战力驱动替代「与谁一起玩」的动机。

### 支柱 3: 双人/多人情境是长期锚点 (2P/Multi as Anchor)
*   **原则**: 陌生人结识后的沉淀，必须落在「固定的人」与「只有我们/小队能玩」的内容上。
*   **意味着 (We Do)**: 预设双人关卡、双人剧情或小队专属情境；挚友/搭档关系有明确的玩法与内容权益。
*   **不意味着 (We Don't)**: 所有内容都设计成单人可完成，导致「有没有固定搭档」无差异；双人内容仅皮肤或数值加成。

## 3. 核心循环 (Macro Core Loop)

```mermaid
graph LR
    旅途攀爬[旅途攀爬] --> 互助协作[互助协作]
    互助协作 --> 结识沉淀[结识与沉淀]
    结识沉淀 --> 挚友关系[挚友关系]
    挚友关系 --> 双人情境[双人情境]
    双人情境 --> 旅途攀爬
```

### 环节解析
*   **旅途攀爬**: 玩家在旅途中攀登山峰，主线路段单人可过，隐藏/分支需协作。
*   **互助协作**: 关键节点依赖托举/拉拽/借力，成功带来过关与关系进度。
*   **结识与沉淀**: 多次协作可建立挚友关系，信任积累。
*   **挚友关系**: 解锁双人/多人情境内容与权益。
*   **双人情境**: 与固定搭档共同体验专属内容，形成「一起登更多山」的新目标。

---

## 4. 关键特性 (Key Features / USP)

1.  **强合作攀岩**: 借鉴 Peak 验证的攀岩+协作，把「互助」做成必须操作、有成败的玩法，而非仅社交行为。
2.  **陌生人→挚友的旅途**: 通过旅途中的互助结识，关系沉淀为挚友，并导向预设的双人/多人情境内容。
3.  **双人/多人情境优先**: 中期后重点提供「只有两人/小队」才能体验的关卡与情境，解决光遇式「缺双人玩法」的困境。
4.  **主线单人可过、协作在隐藏/分支体现**: 保护社恐与独行体验，陌生人互助友善但非必需。

## 5. 目标受众 (Target Audience)

| 层级 | 受众 | 诉求 |
| :--- | :--- | :--- |
| **Core** | 光遇流失/在玩用户 | 要情感与社交，也要「有东西可玩」；渴望真实协作而非仅跑图 |
| **Secondary** | Peak 式合作攀岩轻度玩家 | 通过情感与长期关系转化留存 |
| **Social** | 寻找「陌生人变挚友」「有固定的人一起玩」的社交氛围 | 情感向 MMORPG |
