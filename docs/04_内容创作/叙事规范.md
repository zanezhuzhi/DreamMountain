## [项目名称待定] Narrative Spec / 叙事规范

> **设计原则**：体验优先，结构先行；叙事服务「独行 vs 结伴」的核心张力，避免一切形式的叙事失调。  
> **前置依赖**：本规范建立在已完成并通过评审的世界观文档之上（历史、地理、Biome、环境危害、周目仪式含义、名词术语表），不在此重复定义设定。

---

## 1. 叙事定位（Narrative Positioning）

### 1.1 叙事权重

- **叙事类型定位**：  
  - **玩法驱动 + 氛围叙事** 为主，**强主线叙事为辅**。  
  - 故事主要通过「亲身经历的行为」与「与他人的互动」传达，而非大量文字过场。
- **在整体体验中的权重**：  
  - **主循环层面**：玩家从山脚到山顶的每一次旅程，叙事通过环境变化、他人行动、仪式感行为来嵌入（约 60% 氛围 / 40% 明示）。  
  - **宏观进程层面**：周目、关系升级、解锁双人情境时，通过仪式演出与关键对话承载核心命题（偏向 50% 氛围 / 50% 明示）。

### 1.2 情感基调（Emotional Tone）

- **核心情感关键词**：  
  - **温暖**：被陌生人托起、被无条件帮助后的回温感。  
  - **通透**：对「独行 vs 结伴」两种选择的理解与尊重，而非单向说教。  
  - **可亲近**：场景、角色与交互尽量避免高冷和疏离感，让玩家愿意主动靠近与尝试协作。  
- **避雷方向**：  
  - 避免过度戏剧化的苦难、牺牲与煽情，以免与「日常、可持续的互助体验」冲突。  
  - 避免通过惩罚独行者来强推结伴，情感来源应是「被帮助的真实体感」而非「系统强制」。

## 2. 主题与冲突（Theme & Conflict）

### 2.1 核心命题

- **核心命题**：  
  - 「在一座需要独自面对的高山上，人为什么还会选择彼此托举？」  
  - 延展命题：  
    - 帮助他人时，如果没有任何直接回报，还会有人这么做吗？  
    - 一段关系从「路人」到「挚友」，需要经历怎样的共同体验？

### 2.2 内在张力：独行 vs 结伴

- **价值中立的两端**：  
  - **独行**：专注、自我挑战、自我对话，更高的专注流体验。  
  - **结伴**：协作、照应、情感共鸣，共同完成看似无法独自完成的段落。  
- **冲突来源**（非对立阵营）：  
  - 山的客观难度（环境危害、路线复杂度）让「单人」在某些节点显得捉襟见肘。  
  - 他人的出现与行为，为玩家提供「是否伸出手 / 是否接受帮助」的选择。  
- **叙事目标**：  
  - 不评判玩家是否「必须」结伴，而是让玩家亲身体验：  
    - **被托起那一刻**的情绪差异。  
    - **一起登顶**与「一个人登顶」在意义上的不同。

## 3. 叙事原则（Narrative Principles）

### 3.1 避免叙事失调

- **与玩法奖励的一致性**：  
  - 若叙事宣称「无条件互助」，则玩法层面不得提供明显的数值收益作为直接回报（如立刻获得大量货币、属性提升），以免鼓励「功利性帮助」。  
  - 互助后的反馈以**情感与体验奖励**为主（演出、独特视角、专属情境解锁），物质奖励为辅且具叙事合理性（如「因你的托举，两人共同发现一条捷径」）。  
- **行为语义一致**：  
  - 所有关键行为（托举、拉拽、牵手、献祭、赠与）必须在世界观中有清晰的**语义解释**，并在演出与环境中得到强化。  
  - 严禁出现：系统鼓励「利用他人、抛弃他人」的高效玩法，却口头宣称「真诚互助」。

### 3.2 玩法与叙事融合（Ludonarrative Harmony）

- **设计原则**：  
  - 情感必须由玩家「做出来」，而非「看出来」。  
  - 重要情节点，尽量通过玩家主动操作触发，而非被动观看长过场。  
- **实践要点**：  
  - 所有关键情感节点（Opening/Climax/Twist、周目仪式）都应有：  
    - 明确的**互动动作**（如按住托举、共同蓄力、献祭确认等）。  
    - 与动作紧密绑定的**视听反馈**（镜头、音效、特效、UI）。  
  - 若设计出现「玩家为了通关被迫做出与叙事价值观相反的行为」，必须在本规范的「玩法-叙事融合检查」中记录并调整。

## 4. 叙事手段与资源量级（Methods & Scale）

### 4.1 叙事手段优先级

- **演出（镜头 / 过场 / 特写）**  
  - 用于：Key Moments、周目仪式、关系升级、首次进入双人情境等**里程碑节点**。  
  - 特点：时长短、节奏紧、以「动作与表情」为主，台词克制。  
- **环境叙事**  
  - 用于：路线沿途的前人痕迹、气候变化、地形风格差异。  
  - 特点：持续陪伴型叙事，加强「这座山曾被多少人征服/停留/失败」。  
- **对白与字幕**  
  - 用于：协作提示、关键关系确认、仪式口令、极少量世界观补充。  
  - 特点：简短、口语化、易本地化；可被关闭，不阻断体验。  
- **UI/系统文案**  
  - 用于：关系状态、仪式说明、资源用途解释等高密度信息点。  
  - 特点：清晰准确，不参与抒情，只做信息补全。

### 4.2 资源量级粗估（供排期参考）

> 以下为方向性估算，详细数值在版本立项时进一步 refine。

- **演出 / 过场**  
  - Key Moments：3 段（Opening / Climax / Twist），每段 15–45 秒。  
  - 周目仪式：1 段主流程演出（约 60–90 秒，可拆为数个阶段性镜头）。  
  - 关系升级 / 解锁双人情境：1–2 段短演出（10–20 秒）。  
- **对白 / 台词**  
  - 协作提示与庆祝用语：  
    - 系统预设短句若干组（每种语境 3–5 条可轮换）。  
  - NPC 典型台词：  
    - 周目相关 NPC / 先祖类 NPC，每个 5–10 条短句。  
  - 总字数级别：单语言不超过 2–3 万字，以便轻量本地化。  
- **环境叙事元素**  
  - 通用可复用物件若干（刻痕、旗帜、废弃营地、绳索残段、风铃等），在关卡规范中按密度配置。

## 5. 角色定义（Characters）

### 5.1 玩家角色（旅者）

- **叙事身份**  
  - 名称：**旅者**（无固定姓名，避免强绑定）。  
  - 背景：来自「世界观文档」中定义的大世界之一隅，带着各自的理由来到此山。具体国别/文化属性不在本规范中展开。  
  - 在故事中的定位：  
    - 每个玩家都是一个真实存在的旅者，并非系统生成的幻影。  
    - 山是他们短暂交汇的舞台，真正的主角是「人与人之间的关系」。  

- **欲望 (Want) vs 需求 (Need)**  
  - **Want（表层目标）**：  
    - 「登上这座山」——证明自己、完成某种承诺、获取某种资源（由世界观文档给出）。  
  - **Need（深层需求）**：  
    - 接受他人帮助、学会托举他人，从「只相信自己」到「愿意在关键时刻把后背交给别人」。  

- **个性与投射**  
  - 默认不设置强烈的性格与背景故事，以留白供玩家自我投射。  
  - 个性主要通过玩家的**行为模式**体现：独行者、慢热陪伴者、热情领路人等，由系统标记但不贴标签。  

- **与世界观综述的衔接**  
  - 旅者前史、来到此山的宏大原因（宗教、科技、社会结构）详见《世界观综述》，本规范只引用其结果：  
    - 「每个来到这里的人，都带着某种无法在平地解决的问题。」  

### 5.2 关键 NPC（若有）

- **身份与功能**  
  - **先祖 / 引路人型 NPC**：  
    - 出现在周目仪式空间或山脚营地，对玩家的行为给出象征性反馈。  
    - 在叙事上代表「曾经也走完这条路的人」，而非传统任务发布者。  
  - **兑换 NPC / 仪式守望者**：  
    - 管理高级货币、永久翼等长期资源的转换。  
    - 在叙事上解释「为什么需要献祭 / 为什么会有回馈」。  

- **与旅者的关系**  
  - 非上下级、非神祇式绝对权威，更接近「走在你前面几步的人」。  
  - 不对玩家作出「好坏评判」，只从行为中读出趋势，并通过台词与演出来映照玩家的选择。

- **与竞品（如光遇）差异点**  
  - 先祖类 NPC 不作为「剧情主导者」，而是**演出放大器**：  
    - 在献祭/周目时展示其他旅者的影子、遗留的光点，强化「群像」而非「主角命运」。  
  - 兑换行为不塑造「物欲驱动的养成线」，而强调：  
    - 你用自己的临时能力，换来了更长期的陪伴资格或记忆刻痕。

## 6. 关键体验演出设计（Key Moments）

> 三个 Key Moments 需在玩法状态、演出手段与情感目标上高度一体化。

### 6.1 The Opening：被陌生人托起 / 拉上去

- **情感目标**  
  - 被帮助者：从「紧张 / 无措」到「惊讶 / 被善意击中」。  
  - 帮助者：从「轻微好奇」到「完成一件微小但有意义的事」。  
  - 群体：传达「这里的默认是互助，而非竞争」。

- **演出手段**  
  - **镜头**：  
    - 托举瞬间短暂切至半特写，强调两人角色的接触动作与目光方向。  
    - 登上安全平台后，给出 1–2 秒的轻摇镜头，展示两人的相对站位。  
  - **音效**：  
    - 托举蓄力时的低频聚拢音，与起跳/拉拽成功时的上扬音。  
    - 成功后短暂的环境音静默 0.5 秒，再恢复，制造「被托起的瞬间留白」。  
  - **特效**：  
    - 脚下或手部的微光轨迹，强调「力道传递」而非强力技能。  
  - **字幕 / 提示**：  
    - 仅在第一次出现时浮现极短提示，如：「他伸出了手。」或「你托起了他。」（根据视角不同）。  

- **与玩法的衔接点**  
  - 触发条件：  
    - 新手期关键断崖，单人理论可过但难度极高；若有他人在附近，则大幅简化难度。  
  - 操作逻辑：  
    - 帮助者：长按托举键/拉拽键进入「准备状态」，镜头稍微拉近。  
    - 被帮助者：靠近提示区域，按下接受键，被拉起。  

- **资源量级估算**  
  - 定制动画：托举/拉拽动作若干套（兼顾不同身形与角度）。  
  - 特效：1 套可复用轻量光效。  
  - 音效：2–3 组短音效（蓄力 / 成功 / 落地）。  

### 6.2 The Climax：双人协作登顶

- **情感目标**  
  - 双人：共同完成长程协作后的**情感高潮**与释然。  
  - 单人：一人登顶时的「孤独成就」——不被贬低，但明显缺失某种共鸣。  

- **演出手段**  
  - **镜头**：  
    - 双人登顶：  
      - 最后一步同时踏上山顶边缘，镜头从背后拉升，露出远景。  
      - 短暂环绕两人，停在两人剪影与远方景色的构图上。  
    - 单人登顶：  
      - 镜头更内省，略靠近角色背影，强调「他和世界」。  
  - **音效 / 音乐**：  
    - 双人：主题动机的完全版，加入和声或第二声部，象征「声音变成了复数」。  
    - 单人：使用同一主题但保留部分空白，编制更简约。  
  - **特效**：  
    - 双人：风雪短暂减弱、头顶光晕略增强，两人身上的细微光点互相呼应。  
    - 单人：风声依旧强烈，光效收敛。  
  - **字幕 / UI**：  
    - 双人：可选显示类似「你们一起登上了山顶」的低调字幕，支持关闭。  

- **与玩法的衔接点**  
  - 触发条件：  
    - 两名关系达到一定程度（如互相托举 / 协作次数达到阈值）的玩家，在短时间内相继踏上顶峰区域。  
  - 差异处理：  
    - 双人：解锁合影行为、特殊姿势或双人情境的入口。  
    - 单人：记录为「独行登顶」，在周目或后续演出中被轻描淡写地提及。  

- **资源量级估算**  
  - 镜头脚本：双人版 + 单人版各 1 套。  
  - 音乐：共用主题，不同编曲 2 版。  
  - 特效：1 套可调参数的天气与光照组合。

### 6.3 The Twist：成为挚友、解锁双人情境

- **情感目标**  
  - 将「多次协作的默契」在一个明确时刻**凝结为关系升级**，给玩家「原来我们已经走到这一步了」的惊觉感。  

- **演出手段**  
  - **挚友确认仪式**：  
    - 双方在相对安静的地点（营地、眺望点）进行一个简短互动：互相确认、同时做出某个动作（如伸手 / 点头 / 放置某物）。  
    - 镜头拉近至两人手部或肩部，弱化脸部细节，突出动作。  
  - **解锁时演出**：  
    - 轻微光效从两人之间连成一条线，最终凝固为关系标记（不常驻 UI，主要用于当下演出）。  
    - 背景音乐短暂切入一个温暖的和弦，随后回到环境音。  
  - **首次进入双人情境**：  
    - 进入专属双人区域或玩法时，有一个微型入场演出（5–10 秒）：两人并肩走入场景，镜头像纪录片式跟随。  

- **与玩法的衔接点**  
  - 触发条件：  
    - 累积多次高质量协作（托举、共同通过高风险路段、互相等待等），并在非战斗 / 非高压环境中共同停留。  
  - 系统表现：  
    - 关系状态从「同路人」提升为「挚友」，解锁双人情境入口、共乘道具或专属互动动作。  

- **资源量级估算**  
  - 专用短演出 1–2 段。  
  - 特殊动作或表情若干。  
  - 音乐过门 1 段。

## 7. 周目仪式 / 献祭演出设计（Cycle Ritual）

> 世界观已经定义周目的叙事含义：玩家通过某种形式的献祭或转化，获得长期成长或世界层面的反馈。本节只负责演出与情感节奏。

### 7.1 演出流程

1. **进入仪式空间**  
   - 由主世界中的固定入口进入（山顶 / 特定裂隙 / 神龛），环境音逐渐降低，音乐转为低频引导。  
   - 画面从开放山景慢慢过渡到封闭但通透的仪式空间。  
2. **献祭 / 转化准备**  
   - UI 清晰展示将要失去与获得的内容（如临时翼 → 高级货币 / 永久翼），同时给出简短叙事说明。  
   - 玩家需要主动确认，避免「被系统拖走」。  
3. **执行献祭**  
   - **失去临时翼**：  
     - 翅膀光效一片片化作光点，向仪式空间的中心或天空汇聚。  
     - 对应音效为逐渐收束的高频闪光声。  
   - **获得高级货币 / 永久翼**：  
     - 某种更内敛的光标记出现在角色或界面上，代表「更深层的积累」。  
4. **转化完成与回归**  
   - 镜头短暂远离，展示角色在仪式空间中的小小身影，被光流环绕。  
   - 随后光线收回，玩家回到山脚或某个起点营地，带着新的状态开始下一周目。

### 7.2 情感曲线

- **牺牲感**：  
  - 短暂让玩家体会「失去当下能力」的不舍，通过视觉与音效拉长 2–3 秒。  
- **转化感**：  
  - 通过能量流动、环境响应的变化，让玩家意识到「这些能力并未消失，而是以另一种形式存在」。  
- **重生 / 成长感**：  
  - 回到初始区域时，微妙地改变环境（光线、NPC 台词、可见痕迹），暗示「你已经不是第一次上山的人」。

### 7.3 与光遇暴风眼的差异

- **情感强度**：  
  - 更偏向内省与温暖，而非高压与撕裂。  
- **时长与节奏**：  
  - 整个仪式控制在 60–90 秒内，避免每周目重复时的疲劳。  
  - 允许玩家在熟悉后快速略过部分演出，但保留关键情感节点（失去 / 获得的瞬间）。

### 7.4 资源量级估算

- 场景：1 个主仪式空间，可按阶段变化光照与细节。  
- 音乐：1 条主主题变奏，支持分段触发。  
- 特效：若干组光流、粒子与翼相关动画。  
- 过场：1 套可略过的完整流程演出。

## 8. 环境叙事规范（Environmental Narrative）

### 8.1 环境叙事元素与含义

- **峭壁刻痕**：前人尝试攀爬留下的刻记，密集程度与高度相关。  
- **前人绳索 / 残留锚点**：  
  - 表示常用路线与高危路段；断裂的绳索暗示失败与危险。  
- **营地篝火 / 废弃营地**：  
  - 有火的表示近期有人停留，无火的表示旧迹；附属物件（杯具、书页等）轻度补充情绪。  
- **风铃 / 旗帜 / 祈愿物**：  
  - 在风口或高点，代表不同群体的存在与信念，由世界观文档给出符号含义。

### 8.2 与关卡设计的对应

- **配置规则（示例）**：  
  - 每段主要路线至少 1–2 个「前人痕迹」元素，作为节奏点。  
  - 高危节点必须配置至少一种强提示元素（断绳、密集刻痕、警示旗）。  
- **叙事密度区间**：  
  - 新手区：密度较高，引导玩家理解「这里曾有很多人来过」。  
  - 中期区：密度适中，留出更多给玩家自身体验。  
  - 高难区：环境元素稀疏，强调「前人少有人至」。

### 8.3 无对白 / 无过场的情感传达

- **光照**：  
  - 温暖基调：主色温偏暖，避免极端冷色占主导。  
  - 危险地带：局部使用冷色或高对比光影，但在安全点恢复温暖。  
- **天气**：  
  - 风雪、迷雾强度与玩家当前所处阶段情绪相配合，不与叙事目标矛盾。  
- **场景布局**：  
  - 在容易产生「等待」或「观察」的点安排视野开阔的观景位，让玩家自然停下脚步。

### 8.4 强化世界观解释

- 若世界观对「寒冷 / 风暴 / 高反」等危害已有设定，则：  
  - 视觉：在对应区域统一使用特定视觉语言（色彩、纹理、符号）。  
  - 音效：赋予该危险独特的声学特征，让玩家闭眼也能分辨。

## 9. 对白与台词规范（Dialogue）

### 9.1 对白可见的关系门控

- 与关系系统保持一致：  
  - **路人**：仅可见极少量系统提示性对白，不出现高情感浓度语句。  
  - **同路人**：在多次协作后解锁部分轻量寒暄与鼓励语。  
  - **挚友**：解锁更私密、更深度的台词（可选，避免强干扰）。  

### 9.2 预设对白（若有）

- **协作节点提示**（示例方向）  
  - 「需要帮忙吗？」  
  - 「等我一下，我来托你。」  
- **协作成功后的庆祝用语**  
  - 「我们一起做到了。」  
  - 「要不是你，我可能掉下去了。」  
- **NPC 典型台词**  
  - 围绕「周目意义、山的记忆、人与人之间的托举」展开，避免大量解释世界细节。

### 9.3 对白风格

- **语气**：克制、真诚、略带羞涩，避免过度热血或口号化。  
- **用词**：简短、生活化，便于多语言翻译；不依赖双关或复杂段子。  
- **长度**：大部分对白不超过 1–2 行字幕长度。  
- **多语言支持**：  
  - 所有对白需按「易翻译 / 易本地化」原则编写，避免强文化依赖梗。

### 9.4 无对白设计

- 明确标注哪些场景**禁止对白**：  
  - Opening 被托起瞬间、Climax 登顶前 3 秒、周目仪式牺牲瞬间等。  
  - 这些场景只用演出与氛围承载情绪。

### 9.5 资源量级

- 对白总条数：控制在可维护范围（例如 < 500 条），后续在详细规划中细化。  
- 单语言总字数：建议 < 3 万字。  
- 本地化语种：根据项目规划确定，所有台词需保留简洁原义，方便翻译。

## 10. 情感节拍与结构（Structure & Emotional Beats）

### 10.1 结构框架

- 采用**三幕式结构**，同时映射到 Macro Loop「旅途 → 互助 → 挚友 → 双人情境」。  

### 10.2 关键节拍与玩法对应

| 阶段 | 节拍 | 叙事情境 | 对应玩法 / 流程 |
| --- | --- | --- | --- |
| 开端 | 激励事件 | 第一次真正感受到山的危险与高度 | 新手期首次高难坡 / 断崖 |
| 发展 | 跨越门槛 | 第一次被人托起或托起别人 | Opening Key Moment |
| 发展 | 中点转折 | 意识到「独行很难，但不是不可能」与「结伴更轻松但需要信任」 | 多次协作 / 短暂分离 / 重逢 |
| 高潮 | 终极考验 | 双人协作或独行登顶 | Climax Key Moment |
| 结局 | 回归 / 周目 | 献祭临时能力换取长期成长，开启新一轮 | 周目仪式与新一轮旅程 |

### 10.3 情感曲线（阶段目标）

- **新手期（0–3h）**：好奇、紧张、初步信任。  
- **首周目（约 15h 内）**：  
  - 前半：期待、尝试互助、偶尔挫败。  
  - 后半：稳固信任、形成「固定同路人」或明确选择独行。  
- **中期 / 多周目**：熟练、归属感、对山与他人关系的再理解。  
- **周目仪式节点**：牺牲感→转化感→成长感。

### 10.4 叙事插入节点

- 明确标注需要叙事设计介入的玩法节点（示例）：  
  - 第一次跌落大坑后仍被救起的时刻。  
  - 第一次被别人在安全点等待。  
  - 第一次一起在营地过夜。  
  - 每次周目结束的仪式空间。  
- 这些节点必须有对应的**演出 / 环境变化 / 对白**，而非完全由系统随机生成。

## 11. 玩法–叙事融合检查清单（Ludonarrative Harmony Checklist）

### 11.1 潜在冲突点总表

- **玩法鼓励的行为 vs 叙事主题**  
  - 是否存在「扔下队友更高效」的通关路径？  
  - 是否存在「利用他人作为一次性工具」的最佳解？  
- **资源消耗、关系解锁的叙事解释**  
  - 高级货币 / 永久翼来源与用途是否在叙事上自洽？  
  - 关系升级是否需要「刷行为」，还是自然随真实体验发生？  
- **利他设计与情感目标**  
  - 奖励是否足够「非功利」，让玩家愿意出于情感动机而不是仅出于收益？

### 11.2 关键玩法融合检查（当前版本结论占位）

- **托举**  
  - 叙事意义：把自己的安全和体力部分交给对方，是一种信任动作。  
  - 融合检查：奖励以体验与情境解锁为主，不通过数值强化来诱导滥用。  
- **拉拽**  
  - 叙事意义：把已经滑落或停滞的人重新拉回路径。  
  - 融合检查：避免设计为「拉人送资源」的刷子玩法。  
- **添加好友 / 成为挚友**  
  - 叙事意义：承认一段共同经历，并愿意在未来持续同行。  
  - 融合检查：避免纯数值加成，侧重解锁新体验（双人情境、专属演出）。  
- **周目仪式**  
  - 叙事意义：用一趟旅程中累积的临时能力，换取对未来旅程更深的理解与资格。  
  - 融合检查：确保牺牲感真实存在，但总体体验是积极向上的。
