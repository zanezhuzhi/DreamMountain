# 路径结构说明

本文档说明《代号梦山》项目根目录的推荐划分，将**文档区**与**工程区**明确分离，便于 MkDocs、Cursor 与 UE5 协同使用。

## 推荐目录规划示意图

![《代号梦山》推荐目录规划](../assets/project-structure.png)

## 目录职责

| 路径 | 用途 |
|------|------|
| **`docs/`** | [文档区] 项目文档（原 `项目/代号梦山/`），对标 MkDocs 标准路径，Cursor 会默认在此查找上下文 |
| **`src/DreamMountain/`** | [工程区] UE5 项目根目录，`.uproject` 在此，存放 C++ 代码与 Content |
| **`.github/`** | [同步区] GitHub Actions 自动化脚本 |
| **`mkdocs.yml`** | [同步区] 文档站配置 |
| **`gamespec/`** | [规范区] GameSpec 标准与工具 |
| **`tools/`** | [工具区] Python 等脚本（如 RelationshipManager） |
| **`.gitignore`** | [过滤区] 必须配置，防止 UE 缓存（Intermediate、Saved、Binaries）把 GitHub 塞爆 |

## 迁移说明

- **文档路径**：已将 `项目/代号梦山/` 更名为 `docs/`，符合 MkDocs 和大多数项目的惯例。
- **工程隔离**：`src/` 作为 UE5 隔离带，引擎产生的临时文件（Intermediate/、Saved/、Binaries/）已通过 `.gitignore` 排除，避免 Git 状态栏被数万待提交文件淹没。

> 在 UE5 中新建项目时，请选择 `src/DreamMountain/` 作为项目根目录。

## 新增文档须知

在 `docs/` 下**新增**文档后，需在项目根目录的 `mkdocs.yml` 的 `nav` 中补充对应条目，否则网页侧边栏不会显示。完成后执行 `git add`、`git commit`、`git push origin main`，约 2～5 分钟内网站会自动更新；若未生效，可到 GitHub Actions 手动「Run workflow」触发部署。详见 [网页发布与协作指南](./网页发布与协作指南.md)。
