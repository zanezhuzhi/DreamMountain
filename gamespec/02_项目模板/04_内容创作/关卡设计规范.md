# 关卡设计规范 (Level Design Spec)

> **设计原则**: 关卡是玩法的容器，也是叙事的舞台。好的关卡应该有呼吸感（节奏）和方向感（引导）。

---

### 1.1 拓扑结构 (Topology)
*   **结构类型**: [线性 / 中心辐射 / 银河恶魔城(Metroidvania) / 开放区域]
*   **节点图 (Node Graph)**:
    ```mermaid
    graph LR
        Start --> CombatA
        CombatA --> Puzzle
        Puzzle --> Secret[隐藏房]
        Puzzle --> Boss
    ```

### 1.2 尺度与氛围
*   **空间尺度**:
    *   **主通道宽度**: [例如: 3-5米]
    *   **战斗区面积**: [例如: 20x20米]
*   **引导语言**: 如何指引玩家？（光照、贴图颜色、几何线条）

## 2. 节奏切片 (Pacing Slices)

### 2.1 强度曲线 (Intensity Curve)
描绘玩家在本关卡中的心理压力变化。

*   **起点**: 低压（教学/铺垫）
*   **中段**: 波浪上升（战斗 -> 解谜 -> 战斗）
*   **高潮**: 极高压（Boss战/逃亡）
*   **终点**: 释放（剧情/领奖）

## 3. 实体编排 (Entity Choreography)

### 3.1 遭遇战设计 (Encounter Design)
每一场战斗都必须有战术意图。

*   **Encounter 1**:
    *   **位置**: [坐标/区域名]
    *   **怪物组合**: [怪物A] + [怪物B]
    *   **战术意图**: *示例：利用狭窄地形迫使玩家面对重盾兵，此时远程怪物在后排输出。*

### 3.2 资源投放
*   **补给点**: 必须设置在“高强度战斗前”或“Boss战后”。
*   **隐藏要素**: 探索奖励的位置。

---

## 4. 叙事填充 (Narrative Fills)

关卡本身就在讲故事。

*   **环境叙事**:
    *   *示例：这里的墙壁被炸开了，暗示之前发生过激烈的爆破。*
*   **脚本事件**:
    *   *示例：走到桥头时触发塌陷动画。*
