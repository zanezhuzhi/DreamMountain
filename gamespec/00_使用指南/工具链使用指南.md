# GameSpec 工具链使用指南

> **目标受众**: 负责项目基建的策划、技术美术 (TA) 或程序。
> **核心工具**: `lint_gamespec.py` (静态检查工具)

---\n
## 1. 环境准备

GameSpec 工具链基于 Python 开发。
*   **依赖**: Python 3.8+
*   **库依赖**: 无（使用标准库）

---\n
## 2. 格式检查工具 (Linter)

位于 `../04_自动化工具/lint_gamespec.py`。该工具用于扫描 Markdown 文档，检查是否符合 GameSpec 标准。

### 检查项
工具会自动检查以下错误：
1.  **魔法数字**: 是否直接写死了具体数值（如 `造成 100 点伤害`）。
2.  **必要章节**: 是否缺失了“概述”、“规则”、“验证场景”等核心章节。
3.  **字数限制**: 概述是否过短？描述是否过于冗长？
4.  **关键字**: 规则描述中是否使用了 `必须/MUST` 等标准词汇。

### 使用方法

在终端中运行：

```bash
# 检查单个文件
python bin/gamespec/04_自动化工具/lint_gamespec.py ./项目/我的设计文档.md

# 检查整个目录
python bin/gamespec/04_自动化工具/lint_gamespec.py ./项目/
```

### 输出示例

```text
[FAIL] 07_系统设计_公会.md
  Line 45: Found Magic Number '500'. Please use variable {{GUILD_CREATE_COST}}.
  Line 12: Missing scenario for Rule '加入公会'.
```

## 3. 逻辑验证工具 (Validator)

位于 `../04_自动化工具/validate_logic.py`。
*(高级功能)* 该工具尝试解析文档中的逻辑结构，检查是否存在死循环或未定义的状态流转。

> 注意：此功能依赖于文档中正确使用了 Mermaid 状态机语法。
