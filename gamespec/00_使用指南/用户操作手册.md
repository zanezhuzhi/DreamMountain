# GameSpec 用户操作手册

> **读前必读**: 本手册指导策划人员如何使用 GameSpec 标准库进行日常设计工作。

---
## 1. 快速上手流程 (Workflow)

### 第一步：阅读核心 (Read Core)
**强制步骤**: 在做任何事之前，必须阅读项目根目录下的 **`00_Project_Core/`** (或 `00_项目核心/`)。
*   这是项目的“宪法”。
*   如果你要设计的内容违反了它，请立即停手。

### 第二步：明确需求
在开始动笔前，请先查阅 **[01_规范标准/](../01_规范标准/)** 目录：
*   不懂什么是“好玩”？读 **《玩法设计标准》**。
*   不懂文档格式？读 **《游戏规范标准》**。

### 第三步：选择模板
进入 **[02_项目模板/](../02_项目模板/)** 目录，选择最适合你当前任务的模板：

| 任务类型 | 推荐模板 |
| :--- | :--- |
| **项目初始化** | `00_项目核心/00_核心概念.md` (必选) |
| **新立项/脑暴** | `创意验证书.md` -> `游戏概念书.md` |
| **设计新玩法** | `核心玩法规范.md` |
| **设计新系统** | `系统设计规范.md` (最常用) |
| **写剧情/任务** | `叙事规范.md` 或 `任务规范.md` |
| **配数值表** | `数值配置规范.md` |

### 第四步：复制与重命名
**不要直接修改模板！**
1.  将模板文件复制到你的项目文档目录下。
2.  重命名为 `编号_模块_名称.md` (例如: `07_系统设计_公会.md`)。

### 第五步：填空与设计 (Design)
打开文档，按照模板结构填充内容。

#### A. 数据先行 (Data First)
**不要上来就写规则！** 先思考：这个系统要存什么数据？
*   在 `1.2 数据结构` 中定义 Interface。
*   在 `4.1 配置表` 中定义 Excel 结构。

#### B. 原子规则 (Atomic Rules)
**拒绝小作文！** 将复杂的逻辑拆解为独立的规则卡片。
*   **格式**: 必须包含 Trigger / Check / Execute / Feedback。
*   **技巧**: 推荐使用 `03_智能助手/系统策划助手` 来帮你拆解和生成规则。

#### C. 引用规范
*   **强制项**: 带有 `MUST` 或 `必须` 字样的章节不可删除。
*   **引用项**: 如果涉及数值，必须使用 `{{VAR}}` 格式；如果涉及状态，必须引用 `[机制:XXX]`。

### 第六步：自查与提交
在提交 SVN/Git 之前，请对照文档末尾的 **验证清单 (Checklist)** 进行自查。

## 2. 目录结构说明

GameSpec SDK 包含以下四个核心模块：

*   **01_规范标准**: 我们的“法律”。定义了设计原则、验证逻辑和核心理论。
*   **02_项目模板**: 我们的“图纸”。提供了开箱即用的 Markdown 模板。
*   **03_智能助手**: 我们的“外脑”。包含用于 ChatGPT/Claude 的专家提示词。
*   **04_自动化工具**: 我们的“质检员”。包含自动检查文档格式的 Python 脚本。

### 2.1 自定义空间 (User Space)
GameSpec **允许** 您在项目根目录（甚至 `gamespec/` 内部）创建自定义文件夹。
*   **推荐**: 将草稿、参考资料、非正式文档放在自定义文件夹中（如 `My_Drafts/`）。
*   **注意**: 自动化工具在检查自定义文件夹时会启用**宽容模式**，不强制要求 YAML 格式。

## 3. 常见问题 (FAQ)

**Q: 模板里的章节我不想要怎么办？**
A: 如果是 `必须/MUST` 章节，即便内容为空也必须保留并注明“无”；如果是 `可选/MAY` 章节，可以删除。

**Q: 文档里的链接跳不过去？**
A: 请确保你使用了相对路径。标准库中的引用通常是 `../02_项目模板/xxx`，你需要根据你的文档位置调整层级（例如 `../../gamespec/02_项目模板/xxx`）。
