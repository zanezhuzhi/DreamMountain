# [模板名称] Template / [Name] 模板编写指南

> **用途**: 本文档是 GameSpec 的“图纸”。当你需要创建新的文档模板（如 VR 交互规范）时，请遵循此格式。
> **注意**: 模板本身是给策划填空的，所以要多留 `[ ]` 和说明。

---\n
## 1. 元信息 (Frontmatter)

所有模板必须包含 YAML 头部。

```markdown
---
id: [文档唯一ID]
version: 1.0.0
status: draft
author: [作者]
created: [DATE]
---
```

---\n
## 2. 概述 (Overview)

让策划用一句话讲清楚这个文档是干嘛的。

> **提示**: 在这里写下对策划的指导语。例如：“请简述本系统的核心目标。”

---\n
## 3. 核心内容 (Core Content)

根据业务需求定义章节。必须包含以下结构：

### 3.1 规则定义
*   引导策划使用表格或列表来列出规则。
*   提供示例：`| 规则 | 条件 | 结果 |`

### 3.2 资源需求
*   强制包含 `美术需求` 和 `音效需求` 章节，防止漏提需求。

---\n
## 4. 引用规范 (References)

强制策划使用标准引用格式。

*   **配置表**: 链接到 Excel。
*   **通用机制**: 链接到 `../通用机制库.md`。
