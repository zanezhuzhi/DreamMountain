# 玩法设计标准 / Gameplay Design Standard

> "好玩"不是玄学，而是可拆解、可验证的科学。
> "Fun" is not magic; it's a dissectible, verifiable science.

## 1. 设计哲学 / Design Philosophy

本标准旨在将抽象的"好玩"转化为具体的**设计约束**和**验证标准**。

### 1.1 核心原则 / Core Principles

| 原则 | 说明 | Principle | Description |
|------|------|-----------|-------------|
| **玩家中心** | 设计是为了服务玩家体验，而非展示设计师的聪明 | Player Centric | Design for player experience, not designer ego |
| **即时反馈** | 任何输入都必须有即时且清晰的反馈 | Instant Feedback | Every input MUST have immediate, clear feedback |
| **易学难精** | 低门槛进入，高天花板探索 | Easy to Learn, Hard to Master | Low barrier to entry, high ceiling for mastery |
| **情感驱动** | 机制服务于情感，而非为了机制而机制 | Emotion Driven | Mechanics serve emotions, not just themselves |

## 2. 核心体验架构 / Core Experience Architecture

每个项目**必须**定义其核心体验架构，推荐使用 MDA 框架。

### 2.1 美学目标 (Aesthetics) - 想要传达的情感
项目**必须**明确定义 3-5 个核心情感关键词（如：紧张、放松、探索欲、成就感）。
*示例：PGB项目的关键词是“诙谐”、“软萌”、“治愈”。*

### 2.2 核心循环 (Core Loop)
项目**必须**定义玩家最基础的行为循环。
*   **行动 (Action)**: 玩家做什么？
*   **反馈 (Feedback)**: 世界如何回应？
*   **进展 (Progression)**: 状态如何改变？

### 2.3 验证标准 / Verification
- **必须**在文档中描述核心循环的完整流程。
- **必须**说明核心循环如何强化美学目标。

## 3. 3C与手感标准 / 3C & Game Feel

3C (Character, Camera, Control) 是游戏体验的基础。

### 3.1 控制 (Control)
- **响应性 (Responsiveness)**: 输入延迟**应该**低于 100ms（动作游戏建议 <16ms）。
- **容错性 (Coyote Time)**: **应该**提供边缘容错（如“土狼时间”，离开平台后短时间内仍可跳跃）。
- **输入缓冲 (Input Buffer)**: **应该**支持输入缓冲，避免吞键。

### 3.2 摄像机 (Camera)
- **可见性**: 摄像机**必须**始终保证主角和关键目标在视野内。
- **平滑度**: 移动和旋转**必须**有阻尼和加速，避免生硬的瞬移。
- **遮挡处理**: **必须**有防遮挡机制（如物体透明化或相机推近）。

### 3.3 角色 (Character)
- **重量感**: 移动和跳跃**必须**体现加速度和惯性，避免“飘”在空中。
- **接触感**: 脚步与地面接触**必须**有视觉（灰尘）和听觉（脚步声）反馈。

## 4. 反馈与多汁性 / Feedback & Juiciness

"多汁" (Juicy) 指游戏给予超出预期的丰富反馈。

### 4.1 视觉反馈 (Visual)
- **形变 (Squash & Stretch)**: 角色动作**应该**包含挤压和拉伸，体现弹性。
- **特效 (VFX)**: 关键交互（攻击、跳跃、拾取）**必须**伴随粒子特效。
- **屏幕震动 (Screen Shake)**: 强力冲击**应该**有适度的屏幕震动。
- **顿帧 (Hit Stop)**: 攻击命中瞬间**可以**有微小的停顿（3-10帧）以增强打击感。

### 4.2 听觉反馈 (Audio)
- **层次感**: 声音**应该**分为前景（关键反馈）、中景（环境互动）、背景（氛围）。
- **变化性**: 重复动作的音效**应该**有音高(Pitch)和音量(Volume)的随机微调，避免机械感。

### 4.3 界面反馈 (UI)
- **点击反馈**: 所有可点击元素**必须**有按下、悬停、禁用三种状态的视觉变化。
- **动态效果**: 数值变化**应该**有滚动动画，而非瞬间跳变。

## 5. 心流与节奏 / Flow & Pacing

### 5.1 难度曲线
- **引导期**: 没有任何失败惩罚，专注于教学。
- **挑战期**: 难度阶梯式上升，遵循“挑战-休息-挑战”的波浪式节奏。
- **心流通道**: 难度**必须**动态匹配玩家能力的提升（或提供难度选择）。

### 5.2 动力学 (Motivation - SDT)
- **自主感 (Autonomy)**: 玩家**必须**有 meaningful choices（有意义的选择），而非单一直线。
- **胜任感 (Competence)**: 游戏**必须**清晰地展示玩家的成长（数值、外观、技巧）。
- **归属感 (Relatedness)**: (多人游戏) **应该**设计促进合作的机制，而非单纯的共存。

## 6. 设计验证清单 / Design Verification Checklist

在完成设计文档后，**必须**使用此清单自查：

### 基础体验
- [ ] **核心循环清晰吗？** (能否用一句话描述玩家这1分钟在做什么)
- [ ] **目标明确吗？** (玩家随时知道自己该做什么吗)
- [ ] **反馈即时吗？** (按下按钮后是否立即有反应)

### 3C 手感
- [ ] **摄像机是否碍事？** (是否存在需要玩家频繁手动调整视角的情况)
- [ ] **操作是否顺手？** (常用键位是否符合人体工学)
- [ ] **角色移动自然吗？** (起步、停止、转身是否有惯性细节)

### 反馈质量
- [ ] **视觉特效完整吗？** (交互点是否有高亮/粒子/动画)
- [ ] **音效覆盖全吗？** (每个动作是否有对应音效)
- [ ] **UI 反馈清晰吗？** (点击、获得、错误是否有明确提示)

### 情感与动力
- [ ] **符合美学目标吗？** (当前设计是否传达了预定义的“核心情感”)
- [ ] **失败有挫败感吗？** (失败后是否有“这就再来一次”的冲动，而非想关游戏)
- [ ] **奖励有吸引力吗？** (玩家是否真的想要那个奖励)
